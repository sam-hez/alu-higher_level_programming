# Python - Everything is object

This project is a deep dive into how Python handles objects, focusing on the differences between mutable and immutable types, object identity, and reference sharing.

## Learning Objectives
- Understanding object identity (`id()`) and type checking (`type()`).
- Distinguishing between mutable (e.g., lists) and immutable (e.g., integers, strings, tuples) objects.
- Learning how assignments and aliasing work in Python.
- Grasping the nuances of shallow vs. deep behavior in references.
- Understanding Python's mechanism for passing variables to functions.

## Files
- `0-answer.txt` through `28-answer.txt`: Conceptual answers to object-related questions.
- `19-copy_list.py`: A Python function that returns a shallow copy of a list.
